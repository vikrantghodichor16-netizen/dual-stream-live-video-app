JavaScript
const express = require("express");
const http = require("http");
const cors = require("cors");
const { Server } = require("socket.io");

const app = express();
app.use(cors());

const server = http.createServer(app);

const io = new Server(server, {
  cors: {
    origin: "*",
    methods: ["GET", "POST"]
  }
});

io.on("connection", (socket) => {
  console.log("User connected:", socket.id);

  socket.on("join-room", (roomId) => {
    socket.join(roomId);
    socket.to(roomId).emit("user-joined", socket.id);
  });

  socket.on("offer", ({ roomId, offer }) => {
    socket.to(roomId).emit("offer", { offer, sender: socket.id });
  });

  socket.on("answer", ({ roomId, answer }) => {
    socket.to(roomId).emit("answer", { answer });
  });

  socket.on("ice-candidate", ({ roomId, candidate }) => {
    socket.to(roomId).emit("ice-candidate", { candidate });
  });

  socket.on("disconnect", () => {
    console.log("User disconnected:", socket.id);
  });
});

server.listen(5000, () => {
  console.log("Server running on http://localhost:5000");
});

# Methodology

## Architecture Choice

I selected WebRTC because it is designed for real-time audio/video communication with low latency. It allows media streams to transfer directly between browser peers after connection setup.

I used Socket.IO with Node.js and Express as the backend signaling server. The backend does not process or store video. It only transfers signaling data such as WebRTC offer, answer, and ICE candidates.

React was selected for the frontend because it allows clean routing between the Client Interface and Host Dashboard.

## Backend Architecture

The backend uses Express to create an HTTP server and Socket.IO for real-time communication. Each client and host joins the same room. Socket.IO forwards signaling messages between them.

Actual video is not sent through the backend. This improves performance and reduces server load.

## Challenges Faced

1. Capturing webcam and screen at the same time.
2. Sending two different video feeds to the host.
3. Adding live timestamp permanently into the video.
4. Handling WebRTC offer, answer, and ICE candidate exchange.
5. Managing browser permission issues.

## Solutions

Webcam was captured using getUserMedia and screen was captured using getDisplayMedia.

For timestamp overlay, I used a canvas. The video frame is drawn on canvas and the current time is drawn on top of it. Then canvas.captureStream() converts the canvas into a new video stream.

WebRTC RTCPeerConnection was used to send media tracks to the host. Socket.IO was used only for signaling.

## Final Flow

Client opens the app, grants camera and screen permissions, timestamp is added to the streams, WebRTC sends the streams, and Host Dashboard receives and plays both videos.

import { useRef } from "react";

function ClientPage() {
  const videoRef = useRef();

  const startStreaming = async () => {
    const webcamStream = await navigator.mediaDevices.getUserMedia({
      video: true,
      audio: true,
    });

    const screenStream = await navigator.mediaDevices.getDisplayMedia({
      video: true,
    });

    videoRef.current.srcObject = webcamStream;

    console.log("Webcam Stream:", webcamStream);
    console.log("Screen Stream:", screenStream);
  };

  return (
    <div style={{ padding: "20px" }}>
      <h1>Client Page</h1>

      <button onClick={startStreaming}>
        Start Streaming
      </button>

      <br />
      <br />

      <video
        ref={videoRef}
        autoPlay
        playsInline
        muted
        width="500"
        style={{ border: "2px solid black" }}
      />
    </div>
  );
}

export default ClientPage;
